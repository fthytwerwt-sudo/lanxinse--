# 直播切片 AI Skill 本地解析与兜底样本交付规范

版本：v1.0
用途：客户 / 合作方资料准备说明
状态：`已确认` 本文件只定义资料交付规范；`待验证` AI skill 稳定运行能力仍需后续样本、探针和人工复审验证。

## 0. 先看结论

本项目不是传统固定软件的一次性交付，而是直播切片 AI skill（AI 判断能力 / 自动剪辑能力）建设项目。客户提供资料的目的，不是让我方人工看完或直接解析全部素材，而是让 AI 从客户已有直播录屏、已剪成片、对标视频、字幕文案和业务资料中，反推出稳定的切片判断能力。

本轮核心口径：

- **客户本地解析全量素材，我方不解析全部原始素材**。
- 客户交付给我方的是结构化解析结果、脱敏资料和必要兜底样本。
- 我方留存兜底样本的目的，是用于 AI skill 建设、对照验证、错误纠正、版本横评和验收复盘。
- 兜底样本不是公开传播素材，不用于二次商业使用，除非客户另行授权。
- 资料整理完成不等于 AI skill 已经稳定。稳定能力必须经过多场直播、多类型成片、负样本和盲测验证。

核心路线：

```text
客户本地解析全量素材
-> 生成结构化结果
-> 我方拿到结构化结果 + 必要兜底样本
-> 建设直播切片 AI skill
-> 用兜底样本做对照、纠错、验收
-> 后续再接入 DeepSeek / 其他模型稳定执行
```

## 1. 项目定位

本项目建设的是直播切片 AI skill。这里的 AI skill 指一套可复用的判断能力和自动化执行能力，核心包括：

| 能力 | 中文说明 | 当前状态 |
|---|---|---|
| segment selection | 候选片段识别 | `待验证` |
| clip structure parsing | 成片结构拆解 | `待验证` |
| raw clip matching | 成片到原始直播位置匹配 | `待验证` |
| cut point reasoning | 起止点判断理由 | `待验证` |
| subtitle and copy suggestion | 字幕、标题、文案建议 | `待验证` |
| batch workflow | 批量直播处理流程 | `待验证` |

这不是让客户提交全部原始素材后由我方人工解析，也不是训练大模型。现阶段要做的是建立资料结构、样本标准和验证闭环，让后续 AI 判断有可学习、可对照、可纠错的依据。

## 2. 双方分工边界

| 角色 | 负责事项 | 不负责事项 |
|---|---|---|
| 客户 | 本地保存全量素材；本地完成初步解析；提供结构化结果、脱敏资料、必要兜底样本、业务资料、授权边界；对 AI 输出做人审 | 不需要无边界交付全部原始素材；不需要把未脱敏隐私资料交给我方 |
| 我方 | 搭建直播切片 AI skill；设计成片反推机制、切片判断字段、模型输出结构、自动剪辑流程和验收标准；用兜底样本做对照验证 | 不直接解析客户全部原始素材；不承诺当前稳定运行能力成立；不把样片数量当成稳定验收 |
| AI | 候选片段识别；成片结构拆解；成片到原片位置匹配；钩子、痛点、卖点、信任点、逼单点、异议处理识别；给出起止点建议和结构化报告 | 不替代最终审美判断；不替代业务风险判断；不替代合规判断 |
| 人工 | 最终审美判断、业务风险判断、是否发布判断、低置信度样本复审、禁词和合规兜底、成交策略判断 | 不把未经验证的 AI 结果直接当成稳定能力 |

## 3. 客户本地需要解析哪些资料

### 3.1 全量本地解析资料

以下资料建议客户在本地完成解析和整理，不要求全部原始素材交给我方。

| 资料类型 | 为什么需要 | 本地处理方式 |
|---|---|---|
| 完整直播录屏 | 提供上下文、节奏、铺垫、转化节点和失败片段 | 扫描文件、记录时长、主题、质量、是否有转写 |
| 已剪成片 | 反推什么内容能成为高价值切片 | 记录文件名、类型、主题、成片结构、原片位置 |
| 对标视频 | 学习结构、节奏、字幕、钩子、转化节点和包装方式 | 标注学习什么和不能复制什么 |
| 字幕文件 | 支持文本匹配、结构拆解和风险检查 | 脱敏后提供文本或摘要 |
| 直播转写文本 | 支持成片到原片定位和上下文分析 | 本地生成，脱敏后交付结构化结果 |
| 历史标题和封面文案 | 学习客户已有表达风格 | 归类高转化表达，不提供未授权素材 |
| 历史高转化切片文案 | 反推钩子、痛点、卖点和行动引导 | 脱敏后归入成片结构卡 |
| 业务资料 | 帮助 AI 理解产品、用户痛点、异议和禁区 | 用业务资料填写表填写 |
| 禁词和合规资料 | 降低平台风险和业务风险 | 客户本地确认后提供脱敏版或规则摘要 |

### 3.2 客户交付给我方的结构化结果

客户至少需要交付：

- 直播录屏清单。
- 已剪成片清单。
- 成片原片对应表。
- 成片结构卡。
- 字幕 / 转写结果，需先脱敏。
- 负样本片段表。
- 对标视频拆解表。
- 业务资料填写表。
- 脱敏检查表。
- 授权边界说明。

## 4. 为什么我方仍需留存兜底视频样本

我方不需要持有客户全部原始素材，但必须持有一定数量的直播录屏和已剪成片作为兜底样本。原因如下：

1. AI 判断失败时，需要回看原始上下文。
2. 成片结构卡是否正确，需要用原视频验证。
3. 成片到原片起止点是否准确，需要对照原素材。
4. 后续模型版本升级，需要同一批样本做横向比较。
5. 当客户反馈“不像 / 不对 / 不稳”时，需要共同对照物，避免只靠口头描述。
6. 自动流验收不能只看最终视频，还要看 AI 为什么选这段、为什么从这里切、为什么这里结束。
7. 兜底样本是建设稳定 skill 的验证集，不是公开发布或二次传播素材。

兜底样本的使用范围应写入授权说明：仅用于项目内部的 AI skill 建设、验证、纠错、对照和验收，不用于公开传播或二次商业使用，除非客户另行授权。

## 5. 量化资料标准

### 5.1 三阶段总览

| 阶段 | 用途 | 客户本地解析 | 客户交付给我方 | 我方兜底留存 | 对应率要求 |
|---|---|---|---|---|---|
| 最低启动版 | 第一轮 probe（能力探针 / 初步验证），不能证明稳定运行 | **3 场**完整直播、**30 条**已剪成片、**8 条**对标视频、基础业务资料 | 3 场清单、30 条对应表、30 条结构卡、8 条对标拆解、脱敏字幕、业务资料、脱敏检查、授权说明 | **3 场**完整直播、**30 条**成片、**8 条**对标视频、30 条对应记录、30 条结构卡 | 30 条成片中至少 **24 条**能对应原片，最低 **80%** |
| 推荐稳定版 | 搭建第一版可用 AI skill，做跨场景稳定性验证 | **6 场**完整直播、**60 条**成片、**60 段**负样本、**12-20 条**对标视频、完整业务资料 | 6 场清单、60 条对应表、60 条结构卡、60 段负样本、12-20 条对标拆解、脱敏转写、完整业务资料、授权说明 | **6 场**完整直播、**60 条**成片、**60 段**负样本、**12-20 条**对标视频、**2 场**盲测直播 | 60 条成片中至少 **54 条**能对应原片，最低 **90%**，推荐 **100%** |
| 正式验收版 | 判断 AI skill 是否具备较稳定的批量执行能力 | **10 场**完整直播、**100 条**成片、**100 段**负样本、**20 条**对标视频、**2-3 场**盲测直播 | 10 场清单、100 条对应表、100 条结构卡、100 段负样本、20 条对标拆解、全部脱敏转写、完整业务和合规资料、盲测素材清单 | **10 场**完整直播、**100 条**成片、**100 段**负样本、**20 条**对标视频、**2-3 场**盲测直播 | 100 条成片中至少 **95 条**能对应原片，最低 **95%** |

### 5.2 最低启动版

最低启动版只用于第一轮 probe，目标是判断资料结构是否能支持后续 skill 建设。它不能证明稳定运行。

最低启动版我方需要收到：

- **3 场**完整直播录屏。
- **30 条**已剪成片。
- **30 条**成片与原始直播对应记录。
- **30 条**成片结构卡。
- **8 条**对标视频。
- **8 条**对标视频拆解记录。
- 脱敏字幕 / 转写，至少覆盖 **30 条**成片。
- **1 份**业务资料表。
- **1 份**脱敏说明。
- **1 份**授权边界说明。

### 5.3 推荐稳定版

推荐稳定版用于搭建第一版可用 AI skill，并验证不同直播主题、成片类型、节奏和负样本下的稳定性。

推荐稳定版我方需要收到：

- **6 场**完整直播录屏。
- **60 条**已剪成片。
- **60 条**成片与原始直播对应记录。
- **60 条**成片结构卡。
- **60 段**负样本。
- **12-20 条**对标视频。
- **12-20 条**对标视频拆解记录。
- 脱敏字幕 / 转写，覆盖 60 条成片和对应原片上下文。
- 完整业务资料表。
- 完整脱敏检查表。
- 完整授权边界说明。
- **2 场**盲测直播录屏，尽量不参与前期规则蒸馏。

### 5.4 正式验收版

正式验收版用于判断 AI skill 是否具备较稳定的批量执行能力。正式验收必须包含盲测，不得只看参与规则沉淀的样本。

正式验收版我方需要收到：

- **10 场**完整直播录屏。
- **100 条**已剪成片。
- **100 条**成片与原始直播对应记录。
- **100 条**成片结构卡。
- **100 段**负样本。
- **20 条**对标视频。
- **20 条**对标视频拆解记录。
- 全部脱敏字幕 / 转写。
- 完整业务资料、合规资料、授权资料。
- **2-3 场**盲测直播录屏。

## 6. 成片原片对应要求

每条成片都要尽量对应到原始直播。没有原片对应关系的成片，只能作为风格参考，不能作为 gold sample（金样本 / 标准学习样本）。

必填字段：

| English 字段 | 中文解释 |
|---|---|
| `clip_id` | 成片编号 |
| `clip_file` | 成片文件 |
| `raw_recording_id` | 原始直播编号 |
| `raw_recording_file` | 原始直播文件 |
| `start_timecode` | 原片开始时间 |
| `end_timecode` | 原片结束时间 |
| `exact_or_rough` | 精确或大概 |
| `clip_type` | 切片类型 |
| `core_value` | 核心价值 |
| `human_note` | 人工备注，可选 |
| `desensitized_status` | 脱敏状态 |

如果客户说不清起止点，可以先填写大概时间范围，例如“约第 12 分 30 秒到第 14 分 10 秒”或“大概在第 2 场直播中段”。后续可由本地解析工具通过 audio matching（音频匹配）、transcript matching（字幕 / 文本匹配）和 frame matching（画面帧匹配）进一步反推。

## 7. 成片结构卡要求

每条成片需要生成一张结构卡。结构卡不要求人工全部口述，优先由客户本地 AI 解析自动生成，人工轻量确认。

| English 字段 | 中文解释 |
|---|---|
| `clip_id` | 成片编号 |
| `clip_type` | 成片类型 |
| `opening_hook` | 开头钩子 |
| `pain_point` | 痛点 |
| `selling_point` | 卖点 |
| `trust_point` | 信任点 |
| `objection_handling` | 异议处理 |
| `urgency_or_cta` | 逼单 / 行动引导 |
| `structure_segments` | 结构段落 |
| `cut_start_reason` | 为什么从这里开始 |
| `cut_end_reason` | 为什么这里结束 |
| `discarded_before_reason` | 前面为什么不要 |
| `discarded_after_reason` | 后面为什么不要 |
| `ai_confidence` | AI 置信度 |
| `human_review_status` | 人工复审状态 |

## 8. 负样本要求

负样本是“不适合切片”的片段。它帮助 AI 学会不要切什么，比只看好样本更重要。

生成方式：

| 方式 | 说明 |
|---|---|
| 客户人工提供 | 客户提供不适合切片的片段，并写一句原因 |
| AI 自动反推 | 客户本地解析成片时，同时提取成片前 30-60 秒、成片本体、成片后 30-60 秒，让 AI 反推前后为什么没被剪进去 |

常见负样本类型：

- 铺垫过长。
- 只有聊天没有价值点。
- 信息重复。
- 卖点不完整。
- 钩子不强。
- 上下文缺失。
- 情绪弱。
- 画面或声音质量差。
- 涉及敏感信息。
- 容易违规。
- 不适合作为公开传播内容。

## 9. 对标视频要求

对标视频最低 **8 条**，推荐 **12-20 条**，正式验收 **20 条**。

对标视频只用于学习结构、节奏、字幕、钩子方式、转化节点和包装方式。不用于复制品牌资产、人物形象、界面、文案原句、未授权素材或账号标识。

| English 字段 | 中文解释 |
|---|---|
| `reference_id` | 对标编号 |
| `reference_file` | 对标视频文件 |
| `source_platform` | 来源平台 |
| `reference_type` | 对标类型 |
| `learn_what` | 学习什么 |
| `do_not_copy` | 不能照搬什么 |
| `hook_style` | 钩子方式 |
| `rhythm_style` | 节奏方式 |
| `subtitle_style` | 字幕方式 |
| `cta_style` | 行动引导方式 |
| `notes` | 备注 |

## 10. 脱敏要求

客户提供给我方的结构化资料、字幕、转写、截图说明和兜底样本，都必须先做脱敏检查。脱敏后的资料仍然不是公开素材，只是项目内部资料。

### 10.1 字幕和转写文本

必须脱敏：

- 手机号、微信号、地址、身份证号、订单号。
- 学员真实姓名、客户真实姓名、老师私人联系方式。
- 内部群名、内部链接、支付二维码、私域二维码。
- 真实成交截图中的个人信息。
- 评论区用户名、头像、ID。
- 涉及个体案例的可识别细节。

可保留但建议泛化：

- 价格写成 `[价格区间]`。
- 优惠写成 `[限时福利]`。
- 课程名写成 `[课程A]`。
- 老师名写成 `[老师A]`。
- 学员案例写成 `[学员案例A]`。
- 地区写成 `[某城市]` 或 `[地区A]`。

### 10.2 视频画面

必须检查：

- 直播间账号 ID、抖音号 / 平台号。
- 评论区昵称、头像、ID。
- 私信弹窗、后台数据、订单数据、课程后台、支付页面、二维码。
- 客户头像、学员头像、未授权第三方人物、未授权儿童 / 未成年人画面。
- 未授权品牌 Logo。
- 直播推流软件界面、电脑桌面隐私、浏览器书签、文件路径、通知弹窗。

### 10.3 音频

必须处理：

- 私人电话、微信号、地址。
- 学员真实姓名、客户真实身份。
- 内部价格策略、未公开合作信息、不适合外发的私密对话。

### 10.4 美颜、滤镜、主播肖像和声音

- 美颜、滤镜、灯光、构图、字幕样式可以作为视觉风格参考。
- 不要提供后台美颜参数截图，除非客户确认可共享。
- 如果主播肖像和声音已授权，可以提供原始样本用于项目内部分析。
- 如果未授权，只能提供脱敏版或说明版，不能作为长期样本。
- 不允许提取或保存人脸生物识别模板。
- 不允许把人脸、声音授权写成默认成立，必须由客户明确确认。

### 10.5 业务敏感资料

以下内容必须由客户确认是否允许提供：

- 真实成交数据、投放数据、转化率。
- 客户名单、社群截图、内部培训资料。
- 价格底线、佣金政策、合作合同、未公开课程内容。

如不允许提供，应使用区间、代号、示例化文本或摘要版资料。

## 13. 不能承诺的内容

必须明确：

- 客户完成资料整理，不等于 AI skill 已经稳定。
- **3 场直播 + 30 条成片**只能做初步 probe。
- 稳定执行至少需要多场直播、多类型成片、多负样本和盲测验证。
- AI 负责候选片段、结构拆解、起止点建议、字幕 / 文案建议。
- 人工负责最终审美、人感、业务风险和是否发布。
- 未经客户人审，不写审美通过。
- 未经业务验证，不写业务目标通过。
- 未经批量测试，不写稳定运行。
- 我方留存兜底样本只用于 AI skill 建设、验证、纠错和对照，不用于公开传播或二次商业使用，除非客户另行授权。
