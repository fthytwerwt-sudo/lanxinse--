# 直播切片 AI Skill 资料解析与云盘交付清单

客户执行版
请按本清单在本地解析资料，并将结果上传云盘。

## 1. 先交什么

| 模块 | 必须数量 | 客户要做什么 | 上传给我方什么 |
|---|---:|---|---|
| 完整直播录屏 | 6 场 | 必须解析 6 场完整直播 | 6 场原视频、直播清单、转写或字幕 |
| 已剪成片 | 60 条 | 提供 60 条已剪成片，全部进入自动匹配池 | 60 条成片、成片来源自动匹配表 |
| 成片结构卡 | 60 条 | 每条成片拆清楚为什么能切 | 成片结构卡表 |
| 舍弃片段 / 不要切的片段 | 60 组 | 看成片前后 30-60 秒，说明为什么不要 | 舍弃片段表 |
| 对标视频 | 12-20 条 | 拆解节奏、结构、字幕、钩子 | 对标视频和拆解表 |
| 盲测直播 | 2 场 | 单独预留，不参与前期规则沉淀 | 2 场盲测原视频 |
| 业务资料 | 1 套 | 填写课程、痛点、卖点、禁词 | 已脱敏业务文档 |
| 授权与脱敏说明 | 1 份 | 确认哪些素材可留存、可分析 | 授权说明、脱敏检查表 |

必须记住：

- 客户本地解析，全量素材不需要全部交给我方。
- 我方必须收到云盘资料包后，才能开始做直播切片 AI skill。
- 我方留存视频和成片，只用于兜底、对照、验收、修 skill。
- 资料上传不等于 AI skill 已稳定，稳定能力仍要后续验证。

## 2. 6 场完整直播怎么选

6 场必须这样选：

| 类型 | 数量 | 选择标准 |
|---|---:|---|
| 效果较好 | 2 场 | 成交流程顺、互动好、已有多个优质切片 |
| 普通表现 | 2 场 | 正常直播状态，能代表日常水平 |
| 较乱或难度高 | 2 场 | 节奏乱、跑题多、评论复杂、声音或画面不稳定 |

每场直播必须满足：

- 必须是完整录屏。
- 最好包含直播画面、声音、评论区。
- 每场至少能对应 6-10 条已剪成片。
- 文件名必须能和成片对应。
- 有敏感内容时，先脱敏，或在表格里标清楚。

命名方式：

```text
LS001_日期_主题_效果好.mp4
LS002_日期_主题_效果好.mp4
LS003_日期_主题_普通.mp4
LS004_日期_主题_普通.mp4
LS005_日期_主题_较乱.mp4
LS006_日期_主题_较乱.mp4
```

另外单独预留 2 场盲测直播：

```text
BLIND001_日期_主题.mp4
BLIND002_日期_主题.mp4
```

这 2 场盲测直播不要参与前期规则整理，后面专门用来检查 AI skill 是否真的能处理新直播。

## 3. 60 条成片怎么解析

客户不需要提前知道每条成片来自哪场直播。

客户只需要提供 6 场完整直播录屏和 60 条已剪成片。60 条成片全部进入自动匹配池。如果客户知道某条成片来源，可以填写；如果不知道，统一填写“待自动匹配”。

每条成片必须写清楚：

- 是否知道来源：知道 / 不知道 / 不确定。
- 可能来自哪场直播：知道就写，不知道就写“待自动匹配”。
- 客户大概线索：例如主题、主播话术、画面特征，可不填。
- 这条成片属于哪种类型。
- 开头为什么能抓住人。
- 讲中了什么痛点。
- 讲清了什么卖点。
- 用了什么信任背书。
- 有没有处理用户异议。
- 有没有逼单 / 行动引导。
- 成片前 30-60 秒为什么不要。
- 成片后 30-60 秒为什么不要。
- 是否需要人工复核。

后续会用本地 AI / Trae / 自动解析工具反查来源：

- audio matching（音频匹配）：用成片声音在 6 场直播中找相似片段。
- transcript matching（字幕 / 文本匹配）：用成片字幕或转写文本反查原直播位置。
- frame matching（画面帧匹配）：用画面相似度辅助定位。
- manual review（人工复核）：只复核自动匹配失败或低置信度的片段。

匹配结果分三类：

| 结果 | 处理方式 |
|---|---|
| 已匹配确认 | 系统找到对应直播和起止点，人工确认无误 |
| 低置信度匹配 | 系统找到疑似位置，客户或我方快速确认 |
| 未匹配，仅做风格参考 | 暂时找不到原始位置，只学习风格、字幕、包装、节奏 |

成片类型直接从这里选：

| 类型 | 判断方式 |
|---|---|
| 强钩子型 | 开头冲突强，能立刻吸引停留 |
| 痛点共鸣型 | 重点打用户焦虑、困扰、真实处境 |
| 教学价值型 | 有清晰方法、步骤、知识点 |
| 卖点转化型 | 讲产品 / 课程的具体价值 |
| 信任背书型 | 讲案例、资质、结果、专业度 |
| 异议处理型 | 回答贵不贵、有没有用、适不适合等问题 |
| 逼单成交型 | 有明确行动引导或限时转化 |
| 综合型 | 同时包含多种结构 |

第一轮目标：

- 60 条成片全部进入自动匹配池。
- 系统目标是自动匹配出至少 54 条。
- 这不是客户人工交付要求，客户不需要人工提前完成 54 条对应关系。
- 未匹配成片进入待确认 / 风格参考池 / 需补充素材池，不直接作为核心学习样本。

## 5. 上传前必须检查

字幕 / 文案里必须处理：

- 手机号、微信号、地址、订单号。
- 学员真实姓名、客户真实姓名。
- 私域二维码、支付二维码。
- 真实成交截图里的个人信息。
- 价格底线、内部优惠政策、未公开课程内容。

视频画面里必须检查：

- 直播账号 ID、评论区昵称和头像。
- 后台数据、私信弹窗、支付页面、二维码。
- 电脑桌面隐私、浏览器书签、通知弹窗。
- 未授权人物、未授权品牌 Logo。

音频里必须处理：

- 电话、微信号、地址。
- 学员真实姓名。
- 内部价格策略、未公开合作信息。

美颜 / 肖像 / 声音必须确认：

- 主播肖像是否授权。
- 主播声音是否授权。
- 美颜参数是否允许共享。
- 不允许提取或保存人脸生物识别模板。
- 不允许把声音克隆授权写成默认成立。

最后检查：

| 检查项 | 必须结果 |
|---|---|
| 6 场完整直播 | 已上传 |
| 60 条已剪成片 | 已上传，并全部进入自动匹配池 |
| 成片来源状态 | 已填写“知道 / 不知道 / 不确定”；不知道的写“待自动匹配” |
| 60 条成片结构卡 | 已填写 |
| 60 组舍弃片段 | 已填写 |
| 12-20 条对标视频 | 已上传并拆解 |
| 2 场盲测直播 | 已单独放入盲测目录 |
| 业务资料 | 已脱敏 |
| 授权说明 | 已确认 |
| 云盘链接、提取码、截图、文件清单 | 已发送 |

我方留存视频和成片，只用于直播切片 AI skill 建设、对照、纠错、验收，不用于公开传播或二次商业使用，除非另行授权。
