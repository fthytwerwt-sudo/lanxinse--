## 5. 画面判断维度

| 维度 | Codex 可初判 | 必须人工复核 |
|---|---|---|
| 开头画面是否有停留理由 | 是否有标题、人物、动作、场景或对比 | 是否真的吸引目标用户停留 |
| 画面是否兑现口播 | 关键帧是否出现相关动作 / 场景 | 全片动作是否连续、准确、舒服 |
| 字幕重点是否服务理解 | 字幕是否覆盖关键词 | 字幕节奏、遮挡、语气是否舒服 |
| 动作是否和口播一致 | 画面动作是否与文本主题同向 | 动作标准性、安全性、专业性 |
| 镜头切换是否突兀 | 抽帧能否发现明显黑屏 / 跳变 | 完整节奏和转场手感 |
| 画面前后是否跳 | 关键帧场景是否大幅变化 | 细节跳切和情绪连贯 |
| 语气和表情是否断裂 | 关键帧表情姿态是否极端变化 | 真实语气、停顿、呼吸 |
| 促单时画面是否突兀 | 是否出现黑屏 CTA、强字幕、贴纸 | 是否压迫、像硬卖 |
| 标题 / 封面感 | 是否有显著标题和人物主体 | 是否符合用户审美和平台风格 |
| 审美 / 人感 / 发布判断 | Codex 不判断 | 用户人审 |

## 6. 文本结构和画面剪辑分离规则

| 文本状态 | 画面状态 | 允许结论 |
|---|---|---|
| `text_structure_passed` | `visual_editing_pending_review` | 仅文本候选，待画面复核 |
| `partial` | `visual_probe_initial_observation` | 可进入人工复核或素材池 |
| `failed` | 任意 | 文本淘汰，除非人工认为画面可作素材 |
| `pending_validation` | `visual_source_missing_pending_probe` | blocked / 待补来源 |

禁止结论：

- 文本结构顺，所以画面剪辑通过。
- 抽 3 帧没问题，所以全片节奏通过。
- 画面有动作，所以动作标准通过。
- 字幕看起来清楚，所以业务事实通过。
