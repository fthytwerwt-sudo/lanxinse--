# 初剪三段硬标准

状态：`evidence_derived_standard_completed_pending_user_review（证据反推标准已完成，待用户复核）`
生成时间：2026-06-24

## 0. 总原则

初剪不是把五结构标签拼成一条视频。初剪只看三件事：

1. 开头是否让目标观众知道为什么要继续看。
2. 中间是否真的交付了方法、解释、案例、异议回应或信任。
3. 结尾是否自然收住，而不是没有铺垫就硬卖。

如果开头只有刺激，中间没有兑现，结尾再促单，这条初剪必须返工。初剪通过只表示可交给人工剪映继续处理，不表示最终成片、画面审美、业务事实或发布通过。

## 0.1 必答问题索引

- 什么是真吸引：开头让目标观众知道“这是不是在说我，继续看能得到什么”，并能被中间兑现。
- 什么是假吸引：只有结果愿望、刺激标题、直播热场或动作展示，没有兑现路径。
- 什么是真引流：开头能把观众带到后文的讲解、案例、异议回应或行动路径。
- 吸引和引流的区别：吸引是停留理由，引流是承接路径。
- 什么算交付：中间给出原因、方法、步骤、注意点、案例过程或疑问解释。
- 什么只是水话：只喊口号、动作次数、结果承诺、“很多人有效”，没有条件和依据。
- 什么算收住：前面的内容完成后，用总结、动作完成或轻量行动提醒结束。
- 什么是硬卖：没交付就卖、没信任就卖、疑问没回答就卖，或新增未确认业务承诺。
- 正样片证据：本文件用 `pair_0032 / pair_0001 / pair_0004 / pair_0022 / may7_live` 等证据支撑三段标准。
- 负样片反例：本文件用 `pair_0033 / pair_0030` 等反例说明假吸引、水话、硬卖和过强承诺。
- Codex 判断步骤：开头、中间、结尾均写成可执行检查步骤。
- 人工剪映：每一段都列出画面、字幕、动作、节奏和业务事实复核项。

## A. 开头吸引

### 什么算吸引

开头吸引是目标观众在 3-8 秒内能回答：“这段是不是在说我，继续看能得到什么。”硬条件是：

```text
目标人群/场景 + 具体问题/结果张力 + 继续看的理由
```

例：`pair_0032 00:00:00.000-00:00:04.600` 直接抛“30 岁女性练八爪鱼多久”，后面接工具和练习说明；5 月 7 日直播 `00:00:30-00:00:44` 先说明不是普通健身，而是面向产后、盆底、漏松冷等人群。

### 什么是假吸引

假吸引是让人短暂停住，但不提供后文兑现路径。以下开头必须降级：

- 只有结果愿望：例如 `pair_0033 00:00:00-00:00:10.600` 只抛完美线条和动作愿望。
- 只有动作次数：例如 `pair_0030` 用坐球、前后左右、100 次带结果。
- 只有直播开场：声音测试、欢迎、刷屏、闲聊。
- 只有刺激标题：后文找不到方法、案例或解释。

### 什么算引流

引流是“吸引之后把观众带到下一段”。它必须有出口：讲解、案例、异议回应或行动路径。开头只停留在标题，不算引流。

### 吸引和引流有什么区别

吸引是停留理由；引流是承接路径。吸引可以只在开头成立；引流必须检查后文是否兑现。没有后文兑现的吸引，就是标题党或素材点。

### 开头 3-8 秒必须完成什么

3-8 秒内至少完成三件事：

1. 目标对象：谁需要看。
2. 问题或张力：她卡在哪里。
3. 观看理由：后面会解释、教、证明或给下一步。

如果 3-8 秒内只有“想不想”“要不要”“练多久”，但没有对象或问题，最高 1 分。

### 如果 8-15 秒，必须多完成什么

8-15 秒开头可以更长，但必须多完成“兑现方向”：

- 说明后面会教方法。
- 说明后面会解释原因。
- 说明后面会给案例。
- 说明后面会回答疑问。

超过 15 秒仍在热场、铺情绪或重复标题，默认拖沓，必须压缩或换起点。

### 哪些开头只能算素材，不能做开头

- 单句观点，没有后文承接。
- 结果口号，没有对象和方法。
- 直播欢迎语。
- 商品/课程入口。
- 道具展示但不讲用途。
- 高风险强承诺。

### 哪些开头必须向后接兑现段

- 结果感强的开头必须接方法或案例。
- 痛点强的开头必须接解释或动作。
- 年龄/产后/敏感症状开头必须接边界和人工复核项。
- 商品或工具开头必须接“为什么用、怎么用、注意什么”，不能直接接购买。

### Codex 怎么判断

1. 取开头 `0-15s`，找对象、问题、观看理由。
2. 在后文 `15-60s` 找兑现块。
3. 如果开头和兑现块不是同一问题，标 `transition_break`。
4. 输出 `score_hook`、`hook_claim`、`hook_payoff_ref`、`candidate_decision`。

### 人工剪映怎么复核

人工剪映重点看：标题字幕是否和口播一致、第一秒画面是否有停留理由、是否有回声/噪声、开头是否需要补 0.5-1 秒呼吸、画面是否涉及敏感身体展示。

## B. 中间交付

### 什么算交付

中间交付是观众看完后能得到一个明确答案。答案可以是：

- 为什么会出现这个问题。
- 一个动作怎么做。
- 一个道具怎么用。
- 一个案例为什么可信。
- 一个疑问为什么可以或不可以。
- 下一步为什么合理。

### 什么只是水话

水话是没有增加判断和行动的信息。典型水话：

- “坚持就会好”。
- “很多人都有效”。
- “非常简单”。
- “你一定要练起来”。
- “这个动作很好”。

这些话如果后面没有原因、步骤、案例或边界，只能作为情绪辅助，不能算交付。

### 什么算讲解

讲解必须至少有：

1. 目标：解决什么。
2. 方法：做什么动作或怎么判断。
3. 条件：适合谁、什么时候、用什么工具。
4. 注意点：哪里发力、哪里不要发力、有什么风险。

`pair_0001 00:00:14.960-00:01:10.000` 因为保留站位、方向、感受和呼吸，所以能形成教学块。5 月 7 日直播 `00:05:33-00:05:52` 解释瑜伽球用于找发力感觉，`00:09:32-00:09:56` 给出骨盆左右摆动的执行指令，合起来可作为中间交付的初步样本。

### 什么只是动作次数

只出现“前后 100 次、左右 100 次、转圈 100 次”，但没有原因、发力点、适用人群和注意点，就是动作次数。`pair_0030` 是反例：它有动作和次数，但缺解释和边界。

### 什么算案例支撑

案例支撑必须有对象、背景、过程、反馈和意义归纳。`pair_0004 00:00:05.600-00:01:35.200` 有学员年龄、关系背景、学习动机和反馈，可支撑案例信任。没有过程的“她好了、群里炸了、很多人有效”，只是假信任。

### 什么只是结果口号

结果口号只有结果，没有条件：

- “练一次收一次”。
- “问题没有啦”。
- “一定变好”。
- “很多人都有效”。

结果口号可以作为钩子风险点，但不能当交付。

### 什么算异议回应

异议回应必须先承认疑问，再解释依据。`pair_0023 00:00:00-00:00:29.600` 先问男性按摩是否有用，再解释压力、循环和能力改善逻辑。5 月 7 日直播 `00:04:00-00:04:17` 回答“生了 10 年还可以吗”，但涉及健康效果和适用边界，必须人工复核。

### 什么只是强承诺

强承诺是不解释，只保证。比如“当然有用”“肯定可以”“练了就好”。如果没有依据和边界，就不是异议回应。

### 中间没有交付时，结尾促单为什么会硬

结尾行动必须承接前文价值。中间没有交付时，观众还不知道为什么信、怎么做、是否适合自己，此时让她评论、私信、买课或领资料，就是用行动替代内容。硬卖不是因为出现了行动提醒，而是因为行动提醒前没有价值、信任或解释。

### Codex 怎么判断

1. 找中间结构块：教学、案例、异议、信任任选其一。
2. 检查是否有问题/原因/方法/过程/边界，不足则标 `structure_incomplete`。
3. 检查是否只剩口号、次数、结果承诺，不足则 `material_only`。
4. 若中间不成立，结尾促单自动降级为 `hard_sell_too_early` 或 `manual_review_required`。

### 人工剪映怎么复核

人工剪映看动作画面是否和口播一致、字幕是否能拆成步骤、关键动作有没有被遮挡、节奏是否能跟练、敏感身体表达是否适合公开、业务事实是否需要客户确认。

## C. 结尾收束

### 什么算收住

收住是让观众知道这一段到哪里结束。可以是：

- 动作完成。
- 解释归纳。
- 案例意义归纳。
- 异议回答完。
- 下一步提醒。

收住不等于卖。收住的核心是“前面的内容完成了”。

### 什么算自然行动提醒

自然行动提醒必须满足：

1. 前文已经给过价值。
2. 行动和前文内容一致。
3. 行动短，不新增未确认承诺。
4. 不用压迫语气替代理由。

`pair_0022 00:00:46.920-00:00:55.240` 是可参考证据：前文完成三点判断后，用留言和课堂做收束。

### 什么是硬卖

硬卖是价值没交付就卖，疑问没回答就卖，案例没建立信任就卖，或者结尾新增价格、权益、效果承诺。硬卖的判断不是看有没有“课程/留言/私信”，而是看前面是否已经支撑这个行动。

### 什么是早卖

早卖是促单出现得太早。典型表现：

- 开头 0-10 秒就卖。
- 只有刺激标题后马上卖。
- 只有动作展示后马上卖。
- 异议刚提出，还没解释就卖。

`pair_0033 00:00:00-00:00:10.600` 是当前负样片反例：缺价值和信任铺垫，容易变成结果口号或工具导向。

### 什么是突兀卖

突兀卖是前后主题不接：前面讲动作，后面突然讲课程权益；前面讲案例，后面突然讲价格；前面讲疑问，后面突然让下单。即使卖点本身不强，也会因为承接断裂而失败。

### 什么情况下可以引导评论 / 私信 / 领取资料

可以引导的最低条件：

- 前文已经讲清一个方法或回答一个疑问。
- 观众知道为什么要下一步。
- 行动是轻量入口，不包含未确认业务承诺。
- 字幕只说明下一步，不补新事实。

### 什么情况下不能接课程 / 权益 / 价格

不能接的情况：

- 中间没有交付。
- 案例真实性未确认。
- 效果承诺未确认。
- 价格、权益、退款没有客户确认。
- 涉及健康、敏感人群或专业资质边界。

### Codex 怎么判断

1. 回看结尾前一段是否有价值块。
2. 判断行动类型：评论、私信、关注、领取、课程、购买、价格。
3. 检查是否新增未确认承诺。
4. 输出 `score_conversion` 和 `business_fact_status`；不确定时默认人工复核。

### 人工剪映怎么复核

人工剪映看 CTA 是否压迫、字幕是否过强、转场是否突兀、黑屏或贴纸是否破坏理解、语气是否从教学突然变销售、是否需要删掉价格/权益/效果表达。

## D. 初剪通过标准

### 初剪做到什么算可交给人工剪映

满足以下全部条件，才算 `rough_cut_text_handoff_ready`：

1. 开头有对象、问题、观看理由。
2. 中间至少有一个真实交付块。
3. 结尾能收住或只做轻量行动提醒。
4. 每段有 `source_id / pair_id / start_time / end_time / evidence_ref`。
5. 文本结构状态和画面状态分开写。
6. 风险、业务事实、动作正确性、字幕校正项已列出。

### 初剪哪些问题必须 Codex 返工

以下问题属于 Codex 返工，不该丢给剪映：

- 开头没有兑现块。
- 中间只有水话、口号、动作次数。
- 结尾硬卖、早卖、突兀卖。
- 结构块缺主语、原因、方法、结论。
- 需要虚构字幕才能接上。
- 没有证据引用。
- 强行把 `unmapped_clip` 当五结构。

### 哪些问题属于人工剪映加工

以下问题属于人工剪映加工：

- 字幕错字和断句。
- 拼接点、呼吸点、转场。
- 画面裁切、遮挡、字幕位置。
- 动作画面是否清楚。
- 标题/封面/节奏感。
- 是否保留或弱化 CTA。

### 哪些问题属于客户业务确认

以下问题必须客户确认：

- 价格、权益、退款、课程边界。
- 效果承诺、强功效表达。
- 案例真实性和授权。
- 工具承重、安全性。
- 主播资历、学员数量、专业资质。
- 适用人群和健康边界。

### 初剪不能写成最终成片

初剪通过只表示文本结构和本地剪辑包可进入人工剪映；不能写：

- 画面审美通过。
- 动作正确通过。
- 字幕发布通过。
- 业务事实通过。
- 批量稳定。
- `publish_ready=true`。

最终允许状态只能是：

```text
rough_cut_text_handoff_ready
visual_editing_pending_review
business_fact_status=待客户确认
publish_ready=false
```
