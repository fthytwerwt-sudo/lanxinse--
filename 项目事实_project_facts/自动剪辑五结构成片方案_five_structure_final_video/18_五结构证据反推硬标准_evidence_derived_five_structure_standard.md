## 1. 总硬尺

### 必答问题索引

- 什么是真吸引：目标用户、具体问题、继续看的理由、后文兑现同时成立。
- 什么是假吸引：只有刺激标题、结果愿望、动作展示或强承诺，没有兑现路径。
- 什么是真引流：开头承诺能引到后文讲解、案例、异议回应或行动路径。
- 吸引和引流的区别：吸引让人停下，引流让人继续进入下一段。
- 什么算交付：观众能得到原因、方法、步骤、注意点、案例过程或疑问解释。
- 什么只是水话：只喊口号、动作次数、结果承诺或“很多人有效”，没有条件和依据。
- 什么算收住：前面内容完成后，用总结、动作完成或轻量行动提醒结束。
- 什么是硬卖：没交付就卖、没信任就卖、疑问没回答就卖，或新增未确认业务承诺。
- 正样片证据：本文件每类结构均列 `source_id / pair_id / start_time / end_time / evidence_summary`。
- 负样片反例：本文件每类结构均列 `source_id / pair_id / start_time / end_time / failure_reason`，证据不足处标 `部分成立`。
- Codex 判断步骤：本文件每类结构均写成 1 / 2 / 3 / 4 步。
- 人工剪映：本文件每类结构均保留画面、字幕、动作、节奏和业务事实复核项。

### 什么是真吸引

真吸引不是一句狠话，也不是让人短暂停一下。真吸引必须同时满足四件事：

1. 明确目标用户：观众能在 3-8 秒内知道“这是不是在说我”，例如产后妈妈、50+女性、跑跳漏水人群、想学某个动作的人。
2. 明确未解决问题：不是泛泛说“变好”，而是说清卡点，例如漏、松、冷、找不到盆底发力、担心生了很多年还能不能练。
3. 给出继续看的理由：观众能预期后面会得到方法、解释、案例、步骤或判断，而不是只被刺激词吊住。
4. 后文有兑现：开头提出的问题，后面必须接到讲解、案例、异议回应或行动路径。

如果只有目标用户和刺激词，没有后文兑现，只能算假吸引；如果只有标题结果，没有方法方向，只能算标题党；如果只有场面热闹或直播互动，不能算结构开头。

### 什么是假吸引

假吸引是“看起来能停一下，但不能把观众带到有效内容”。满足以下任一项，就不得写 `traffic_hook passed`：

- 只有结果愿望，例如“想要拥有完美线条”，但不知道适合谁、怎么做、为什么可行。
- 只有强承诺，例如“练一次收一次”“问题没有啦”，但没有解释条件、边界和方法。
- 只有刺激标题，没有后续兑现时间码。
- 只有动作展示，没有对象、问题、动作条件和注意点。
- 只有直播开场、问候、声音测试、互动刷屏，没有结构功能。

### 什么是真引流

真引流 `traffic_hook` 比真吸引多一层任务：它不只让观众停下，还要把观众引到后面的结构。硬条件是：

```text
如果出现 目标用户 A + 具体问题 B + 结果/收益预期 C，
并且后文 D 能用讲解/案例/异议/行动兑现这个预期，
才算真引流。

如果只有 A/B/C，没有 D，只能算刺激点或素材点。
```

### 吸引和引流的区别

吸引解决“观众为什么停下来”；引流解决“观众停下来以后为什么继续看下一段”。吸引可以是一句开头理由；引流必须有前后承接关系。吸引没有兑现，最多是标题；引流没有后文，必定失败。

### 什么算中间交付

中间交付必须让观众拿到一个可理解的东西：原因、方法、步骤、注意点、案例过程、异议解释或边界。只喊结果、只给动作次数、只说“坚持练”“很多人有效”，都不算交付。

### 什么只是水话

水话是听起来像内容，但观众看完不能回答“为什么、怎么做、对谁有效、注意什么”。典型表现：

- 重复口号，不增加条件或步骤。
- 重复动作次数，不讲站位、发力、方向、感受或错误点。
- 说“当然有用”，但不给机制解释。
- 说“很多人有效”，但没有对象、背景、过程、反馈。
- 说“留言/领取/课程”，但前面没有交付价值。

### 什么算结尾收束

收束是把前面的内容停在一个观众能理解的位置。自然收束可以是总结、动作完成、下一步提醒、评论/私信入口。硬卖是没有交付就卖、没有信任就卖、用新权益/价格/确定性效果压过内容，或在观众疑问还没被回答时强推行动。

## traffic_hook（引流钩子）

### 1. 大白话定义

`traffic_hook` 是开头把目标观众拉进来，并明确告诉她“接下来值得看什么”的结构。它不是标题文案，也不是一句结果口号；它必须让后文有东西可接。

### 2. 真标准

满足以下全部条件才算真引流：

1. `对象清楚`：3-8 秒内能识别目标用户或目标问题。
2. `问题具体`：不是泛泛变好，而是漏、松、冷、盆底、年龄、生育年限、练多久、怎么练等具体卡点。
3. `承诺可兑现`：承诺必须指向后文能出现的方法、工具、案例或解释。
4. `后文接得住`：同一条视频或同一初剪序列里必须有兑现块，不能只靠字幕编造。

5 分：3-8 秒内对象、问题、观看理由成立，8-15 秒内能看到兑现方向。
3 分：对象和问题成立，但需要后文扩展或桥接。
1 分：有刺激词，但对象、兑现方向缺失。
0 分：标题党、强结果、无后文兑现。

### 3. 假标准 / 常见误判

- 把“想要拥有某结果吗”当钩子，但没有说明人群、条件、方法。
- 把“练一次收一次”当钩子，但后文只有次数和口令。
- 把直播间欢迎语、声音测试、刷屏互动当开头。
- 把单句观点当完整钩子，后面没有解释。
- 把高风险效果承诺当吸引点，未标 `business_fact_status=待客户确认`。

## course_teaching（课程讲解）

### 1. 大白话定义

`course_teaching` 是观众看完后知道“为什么这样做、怎么做、做到什么感觉、哪里容易错”的中间交付。它不是口令集合，也不是动作次数堆叠。

### 2. 真标准

真讲解必须满足至少三类信息：

1. 问题或目标：为什么要练这一段。
2. 方法或步骤：站位、方向、节奏、呼吸、发力、道具、顺序。
3. 感受或注意点：哪里发力、哪里不要发力、做错会怎样、什么人需要谨慎。

完整教学通常 20-60 秒；8-20 秒可作为短教学，但必须至少有问题 + 方法 + 注意点。少于 8 秒默认不是完整教学，除非前后结构已补足。

### 3. 假标准 / 常见误判

- 只有“左右 100 次、转圈 100 次”，没有为什么和怎么做。
- 只有“坚持练就好”，没有发力位置和错误点。
- 只有结果承诺，没有动作条件。
- 只有道具介绍，没有方法。
- 只有主播资历，没有观众可执行步骤。

## objection_response（异议回应）

### 1. 大白话定义

`objection_response` 是把观众心里的疑问摊开，再用解释、依据和边界回答它。它不是用“当然有用”“肯定可以”“很多人都好了”把疑问压下去。

### 2. 真标准

真异议回应必须有：

1. 明确疑问：例如“生了 10 年还可以吗”“按摩真的有用吗”“为什么练了没感觉”。
2. 正面回答：先回应问题，而不是立刻卖课程或推工具。
3. 解释依据：机制、经验边界、身体逻辑、过程说明或案例证据。
4. 适用边界：什么情况下可试，什么情况下要人工/客户/专业确认。

### 3. 假标准 / 常见误判

- 只说“当然可以”，没有条件和边界。
- 用“有效”“有用”“练一次收一次”替代解释。
- 观众疑问还没回答，就直接引导买课、领资料、下单。
- 只讲主播资历，不回答观众疑问。

## case_trust（案例信任）

### 1. 大白话定义

`case_trust` 是用一个具体对象的背景、过程和反馈，让观众知道“这不是主播空说”。它不是“很多人有效”，也不是截一条结果评论。

### 2. 真标准

真案例信任必须有：

1. 对象：谁，年龄、状态、关系或问题是什么。
2. 背景：为什么这个人会进入这个问题或训练。
3. 过程：她做了什么、怎么学、怎么练、发生了什么变化。
4. 反馈：出现了什么反馈或结果。
5. 意义归纳：这个案例说明什么，不强行外推到所有人。

### 3. 假标准 / 常见误判

- 只说“很多人有效”，没有对象。
- 只给结果，没有背景和过程。
- 只讲群里炸了、反馈很好，但没有可验证内容。
- 把未授权案例写成已确认事实。
- 用案例直接推价格、权益或课程效果。

## conversion_push（促单转化）

### 1. 大白话定义

`conversion_push` 是在观众已经得到价值、信任或疑问回应后，告诉她下一步怎么行动。它不是一上来卖，也不是把价格、权益、效果承诺塞到结尾。

### 2. 真标准

自然促单必须满足：

1. 前文已经交付：有讲解、案例、异议回应或明确收束。
2. 行动只做下一步：评论、私信、关注、领取资料、进入课堂等。
3. 不新增未确认承诺：不在结尾突然补价格、权益、退款、确定性效果。
4. 语气承接前文：像“下一步”，不是“现在必须买”。

### 3. 假标准 / 常见误判

- 开头直接卖。
- 中间没讲清就卖。
- 异议没回答就卖。
- 用新权益、新价格、新强承诺强行推。
- 用黑屏 CTA、贴纸、强字幕压过内容，画面待复核时不能写通过。

4. 输出候选决策：促单默认至少 `manual_review_required`，不得直接写发布通过。

不通过：早卖、硬卖、突兀卖、用承诺压过疑问。

## unmapped_clip（无法映射片段）

### 1. 大白话定义

`unmapped_clip` 是安全刹车：当片段不能被证据支撑为五结构之一时，宁可标无法映射，也不要为了凑五结构强行归类。

### 2. 真标准

以下情况应写 `unmapped_clip`：

- 纯音乐、歌词、无关画面、异常片段。
- 只有单句观点，没有结构展开。
- 文本和五结构任务不匹配。
- 缺上下文，向前/向后扩展也无法成立。
- 风险过高，必须先业务确认。

### 3. 假标准 / 常见误判

- 看到动作就归为课程讲解。
- 看到结果就归为引流钩子。
- 看到评论/私信就归为转化。
- 为了让初剪完整，把不可用片段硬塞进中段或结尾。
